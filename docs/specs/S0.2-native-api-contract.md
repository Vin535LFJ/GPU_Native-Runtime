# S0.2 Native API Contract

## 目标

定义 SDK 内部 Kotlin Runtime 与 C++ RuntimeCore 之间的稳定 API 契约，确保后续渲染、资源、Timeline、Audio、诊断模块可以独立实现。业务 App 不直接调用本层 API。

## 范围内

- SDK facade 到 Kotlin runtime 的边界说明。
- Runtime 生命周期 API。
- Clip/Asset 生命周期 API。
- 播放控制 API。
- 渲染 tick API。
- 诊断指标 API。
- 错误码与状态机原则。

## 范围外

- 不定义业务 SDK facade；业务 API 见 `S0.4-sdk-product-contract.md`。
- 不实现 JNI 代码。
- 不定义完整二进制资源格式。
- 不实现具体 GL 渲染逻辑。

## Kotlin Runtime Bridge 草案

```kotlin
class NativeBridge {
    external fun initRuntime(width: Int, height: Int): Long
    external fun resize(runtime: Long, width: Int, height: Int)
    external fun loadClip(runtime: Long, clipPath: String): Int
    external fun unloadClip(runtime: Long, clipId: Int)
    external fun play(runtime: Long, clipId: Int)
    external fun pause(runtime: Long)
    external fun seekTo(runtime: Long, positionUs: Long)
    external fun drawFrame(runtime: Long, vsyncTimeNs: Long)
    external fun getFrameStats(runtime: Long): FrameStatsSnapshot
    external fun releaseRuntime(runtime: Long)
}
```

## C++ RuntimeCore 草案

```cpp
class RuntimeCore {
public:
    bool init(int width, int height);
    void resize(int width, int height);
    int loadClip(const std::string& clipPath);
    void unloadClip(int clipId);
    void play(int clipId);
    void pause();
    void seekTo(uint64_t positionUs);
    void drawFrame(uint64_t vsyncTimeNs);
    FrameStatsSnapshot getFrameStats() const;
    void release();
};
```

## 状态机

```text
Uninitialized → Initialized → Loading → Ready → Playing
                                  ↘ Paused ↔ Playing
Any state → Released
```

## 错误处理原则

- JNI 边界不抛出未捕获 native 异常。
- 可恢复错误返回错误码并写诊断日志。
- 资源加载失败不得导致 runtime 崩溃。
- `releaseRuntime` 必须幂等。

## 验收标准

- 业务 sample 只调用 SDK facade，不直接调用本 NativeBridge。
- Wave 1 工程骨架可按本 API 创建 Kotlin/C++ 空实现。
- 后续功能扩展优先新增细分 API，不破坏生命周期 API。
