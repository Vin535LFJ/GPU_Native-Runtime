# S0.1 Engine Decisions

## 目标

在正式编码前锁定 PetCat Runtime 的第一版工程技术栈、目录边界与关键架构决策，避免后续 Spec 反复修改基础工程。

## 范围内

- Android/Kotlin/NDK/CMake/OpenGL ES 的版本决策。
- 首版渲染宿主选择。
- Kotlin 层、JNI 层、C++ 层的职责边界。
- Debug fallback 与正式资源路径的隔离原则。

## 范围外

- 不实现 Android 工程代码。
- 不接入 BasisU/KTX2 真实解码。
- 不实现 shader、timeline、audio 逻辑。

## 决策

| 项目 | 第一版决策 | 说明 |
| --- | --- | --- |
| 平台 | Android | 当前项目先聚焦 Android Runtime。 |
| 渲染 API | OpenGL ES 3.0 | 满足 2D atlas、多层合成、FBO、基础 FX，首版不并行实现 Vulkan。 |
| 渲染宿主 | `GLSurfaceView` | MVP 阶段降低 EGL 管理复杂度，后续可抽象自定义 EGL。 |
| Native 语言 | C++17 | 用 RAII 管理 GL/native 资源。 |
| Kotlin 职责 | 生命周期、UI、调度、JNI facade | 不承载核心渲染与资源生命周期。 |
| C++ 职责 | RuntimeCore、render、texture、timeline、material、metrics | 保持核心 runtime 可测试、可复用。 |
| 资源目标格式 | KTX2 + BasisU | Debug 可有隔离 fallback，但核心接口面向 GPU-ready texture。 |
| 主时钟 | Audio master clock | 无音频时使用 monotonic clock fallback。 |

## 验收标准

- 后续 Spec 均引用本决策，不再重复讨论首版是否使用 GLES/Vulkan、GLSurfaceView/自定义 EGL。
- 工程骨架 Spec 可以基于本决策直接列出文件结构和依赖。
