# S0.4 SDK Product Contract

## 目标

定义业务 App 可直接调用的 PetCat Android SDK 契约，明确 SDK facade 与内部 runtime/native API 的边界。

## 范围内

- SDK 对外核心类。
- 业务调用生命周期。
- 回调、错误码、配置项。
- SDK facade 到 Kotlin runtime 的职责分配。

## 范围外

- 不实现 SDK 代码。
- 不实现具体 GL 渲染。
- 不定义最终 Gradle 发布流程。

## 对外核心类草案

```kotlin
class PetCatPlayerView : FrameLayout {
    fun bind(player: PetCatPlayer)
    fun unbind()
}

class PetCatPlayer {
    fun load(clip: PetCatClip, config: PetCatConfig = PetCatConfig())
    fun play()
    fun pause()
    fun seekTo(positionUs: Long)
    fun release()
    fun setCallback(callback: PetCatCallback?)
}

data class PetCatClip(
    val uri: Uri,
    val name: String? = null
)

data class PetCatConfig(
    val quality: PetCatQuality = PetCatQuality.Auto,
    val enableFx: Boolean = true,
    val enableStats: Boolean = false
)
```

## Callback 草案

```kotlin
interface PetCatCallback {
    fun onReady(durationUs: Long) {}
    fun onPlayStateChanged(state: PetCatPlayState) {}
    fun onError(error: PetCatError) {}
    fun onStats(stats: PetCatStats) {}
}
```

## 业务调用流程

```text
Activity/Fragment
   ↓
create PetCatPlayerView
   ↓
create PetCatPlayer
   ↓
playerView.bind(player)
   ↓
player.load(PetCatClip(...), PetCatConfig(...))
   ↓
player.play/pause/seekTo
   ↓
player.release + playerView.unbind
```

## 错误码分类

| 分类 | 示例 |
| --- | --- |
| Resource | clip 不存在、schema 不兼容、atlas 缺失。 |
| Device | OpenGL ES 版本不足、纹理尺寸超限、压缩格式不支持。 |
| Runtime | native init 失败、GL context lost、播放状态非法。 |
| Audio | 音频加载失败、audio clock 不可用。 |

## 设计约束

- 业务层不直接调用 `NativeBridge`。
- 业务层不直接管理 GL texture、shader、FBO。
- SDK API 必须对 Activity/Fragment 生命周期友好。
- SDK 内部可以使用 Runtime/Engine 命名，但对外文档应以 SDK 集成为主。

## 验收标准

- Wave 1 的 sample-app 可以只依赖 SDK facade 完成最小集成。
- 后续 Native API 变化不应影响业务 SDK API，除非显式升级 SDK 主版本。
