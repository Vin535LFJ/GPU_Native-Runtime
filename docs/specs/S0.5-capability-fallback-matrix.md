# S0.5 Capability & Fallback Matrix

## 目标

定义 GPU 能力分层（Tier-0/1/2）、功能开关、降级路径与错误码映射，避免“仅检测 extension 但无法稳定决策”的实现风险。

## 范围内

- 设备能力探测字段与分层规则。
- 每个 Tier 的功能白名单/上限。
- 降级顺序与触发条件。
- 错误码与诊断字段约定。

## 范围外

- 不实现具体 capability probe 代码。
- 不实现 shader/fx 具体算法。

## Capability 探测字段（最小集）

- GL 版本：`gl_version_major/minor`（要求至少 GLES 3.0）。
- 纹理上限：`max_texture_size`。
- 压缩格式：ASTC / ETC2 支持位。
- 渲染能力：`max_color_attachments`、`max_draw_buffers`。
- 着色器上限：`max_fragment_uniform_vectors`。
- GPU 时间查询可用性：timer query 支持位（用于 `gpu_frame_ms`）。

## Tier 定义（建议）

### Tier-0（最低可运行档）

- 条件：满足 GLES 3.0，但压缩格式/纹理上限较弱。
- 默认策略：
  - 纹理优先 ETC2 或更保守格式。
  - `pageFrames` 优先 1 或 2；`maxResidentPages <= 2`。
  - 禁用高阶 FX（Bloom/LUT/RimLight）。
- 目标：保证可播放、可观测、可回退。

### Tier-1（中档默认）

- 条件：GLES 3.0 + 稳定压缩纹理支持 + 中等纹理上限。
- 默认策略：
  - `pageFrames = 2`，`maxResidentPages = 3`。
  - 开启 Feather/Shadow 等基础 FX。
- 目标：1080P 主路径稳定。

### Tier-2（高档增强）

- 条件：纹理上限充足、压缩格式能力强、GPU 性能高。
- 默认策略：
  - 可尝试 4K 或更高阶 FX 组合。
  - 可实验 `pageFrames = 4`（需压测门槛）。
- 目标：高画质 + 高分辨率。

## 降级路径（固定顺序）

1. 降分辨率资源档位（4K -> 1080P -> 720P）。
2. 缩减分页缓存（`maxResidentPages` 下调）。
3. 降低每页帧数（2/4 -> 1）。
4. 关闭高阶 FX（Bloom/LUT/RimLight）。
5. 关闭基础可选 FX（Feather/Shadow 可降级）。
6. 若仍不满足，返回明确不支持错误码。

## 错误码建议

- `E_GL_VERSION_UNSUPPORTED`
- `E_TEXTURE_SIZE_LIMIT`
- `E_COMPRESSED_FORMAT_UNSUPPORTED`
- `E_FBO_CAPABILITY_INSUFFICIENT`
- `E_RUNTIME_TIER_DOWNGRADED`（非致命，告警）

## 验收标准

- S1/S2 初始化日志可输出 capability 快照与最终 Tier。
- 每次降级都必须输出“触发原因 + 前后配置 + 错误码/告警码”。
- sample-app 可展示当前 Tier 与降级计数。
