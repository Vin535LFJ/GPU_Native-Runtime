# S0.3 Asset Schema

## 目标

定义 PetCat clip 的第一版资源描述 schema，使 runtime、petcat-builder、测试资产使用一致的数据契约。

## 范围内

- `clip.json` 顶层描述。
- `timeline.anim` 逻辑字段。
- `material.json` 逻辑字段。
- schema version 与兼容策略。

## 范围外

- 不实现 schema parser。
- 不实现 petcat-builder。
- 不规定最终二进制 `.anim` 编码，MVP 可先使用 JSON 表达同等字段。

## clip.json 草案

```json
{
  "schemaVersion": 1,
  "name": "idle",
  "width": 1280,
  "height": 720,
  "fps": 30,
  "durationUs": 3000000,
  "atlases": [
    {
      "id": "atlas_0",
      "path": "atlas_0.ktx2",
      "format": "ktx2-basisu",
      "width": 2048,
      "height": 2048
    }
  ],
  "timeline": "timeline.anim",
  "material": "material.json",
  "audio": "idle.audio"
}
```

## timeline.anim 逻辑字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `schemaVersion` | int | timeline schema 版本。 |
| `durationUs` | long | clip 时长。 |
| `tracks` | array | 多轨道定义。 |
| `layers` | array | layer 定义和 z-order。 |
| `frames` | array | frame 到 atlas rect 的映射。 |
| `events` | array | 可选事件点。 |

## material.json 逻辑字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `schemaVersion` | int | material schema 版本。 |
| `materials` | array | material 列表。 |
| `shader` | string | shader 名称或路径。 |
| `blendMode` | string | `normal`、`premultipliedAlpha`、`additive` 等。 |
| `uniforms` | object | uniform 默认值。 |
| `textureSlots` | object | 纹理槽位绑定。 |

## 兼容策略

- Runtime 只保证读取当前主版本 schema。
- 增加可选字段不提升主版本。
- 删除/重命名字段必须提升主版本。
- 所有 parser 必须输出清晰错误路径，例如 `clip.atlases[0].path missing`。

## 验收标准

- petcat-builder MVP 和 runtime loader MVP 使用同一套字段命名。
- 示例资源可表达单层、多层、带音频、无音频四类 clip。


## 帧分页与合图策略（新增决策）

### 1) 每页帧数建议

首版默认采用 **2×1（每页 2 帧）**，原因：

- 相比 1×1，IO/解码切页频率约减半。
- 相比 4×1，单页显存压力与翻页突刺更可控。
- 对帧动画场景可在“预加载灵活性”和“吞吐效率”间取得平衡。

工程建议：

- `pageFrames = 2` 作为默认值。
- 低内存设备允许降到 `pageFrames = 1`。
- 高端设备或低帧率长镜头可实验 `pageFrames = 4`（需单独压测）。

### 2) KTX2 与 Texture Array 的使用建议

- KTX2 可承载 array texture 数据；OpenGL ES 3.0 也支持 `sampler2DArray`。
- 但帧动画首版推荐 **Atlas + UV 裁剪**，暂不作为默认主路径启用 texture array。

原因：

- Shader 与材质系统复杂度更低，便于先打通 MVP。
- 现有 layer/material 语义与 atlas rect 映射天然匹配。
- 诊断与回退策略（换页、预取、LRU）更直观。

### 3) 显存预算与缓存页建议

为避免“算不清/超预算”问题，`clip.json` 对应 runtime 需要显式预算参数（可在后续 schema 细化中落字段）：

- `pageFrames`：每页帧数，默认 2。
- `prefetchPages`：预取页数，默认 1（前后窗口另算）。
- `maxResidentPages`：缓存上限页数，默认 3。

1080P（ETC2）建议：

- 2×1 单页约 4.2MB，3 页约 12.6MB。
- 对 2GB RAM 低端机可接受，但建议按设备能力动态收紧到 2 页，并允许降到 1×1。

4K（ASTC 4x4）建议：

- 2×1 单页约 13.2MB，3 页约 39.6MB。
- 仅在高端设备档位开放，且需与 FX 档位联动，避免与后处理纹理叠加导致峰值过高。

### 4) 设备分档策略（用于 S4/S8）

- Low（2GB RAM 级）：优先 1×1 或 2×1 + `maxResidentPages<=2`，默认关高阶 FX。
- Mid：默认 2×1 + `maxResidentPages=3`。
- High：可试 4×1 或 4K 2×1，但必须通过机型白名单和压测门槛。
