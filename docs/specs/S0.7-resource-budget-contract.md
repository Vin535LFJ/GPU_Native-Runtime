# S0.7 Resource Budget Contract

## 目标

定义显存预算、上传带宽预算、缓存淘汰阈值与优先级，避免“功能可跑但实机抖动”。

## 范围内

- 预算模型（按设备档位/分辨率）。
- 每帧上传预算与突发上限。
- eviction 触发阈值与优先级。
- 预算相关 QoS 指标。

## 范围外

- 不实现具体 TextureCache 代码。
- 不绑定单一平台 profiler 工具。

## 预算模型（建议）

运行时预算至少包含：

- `gpu_budget_mb`
- `upload_budget_mb_per_frame`
- `upload_burst_mb`
- `max_resident_pages`
- `fx_budget_mb`

## 设备档位默认值（首版建议）

- Low：更小 `gpu_budget_mb`，`max_resident_pages<=2`，高阶 FX 默认关闭。
- Mid：`max_resident_pages=3`，`pageFrames=2`。
- High：可开放更高分辨率与更多 FX，但受总预算硬阈值保护。

## 上传预算协议

- 每帧上传量超过 `upload_budget_mb_per_frame` 时，必须切片到后续帧。
- 单次突发不得超过 `upload_burst_mb`。
- 上传任务按优先级队列执行：当前帧 > 下一帧窗口 > 远期预取。

## eviction 触发阈值

- Soft watermark：达到预算 85% 开始机会性回收。
- Hard watermark：达到预算 100% 强制回收非活跃资源。
- Emergency：超过预算且无可回收对象时触发降级（降页/降分辨率/降 FX）。

## eviction 优先级

1. 最久未使用且不在播放窗口资源。
2. 远期预取页。
3. 可重建中间缓存（如可重复生成的中间 FBO，若策略允许）。
4. 严禁回收当前帧必需资源。

## QoS 指标（必须）

- `late_frame_ratio`
- `upload_stall_ms`
- `cache_hit_ratio`
- `memory_watermark_mb`
- `fallback_trigger_count`

## 验收标准

- S4 阶段可输出预算快照与水位事件日志。
- 触发 hard watermark 时，无崩溃且可观测到降级路径。
- QoS 指标可用于回归对比（版本间可比较）。
